Contexto do Problema

O desafio

Em ambientes de marketing digital multi-canal, um cliente raramente converte no primeiro ponto de contato. Ele pode ver um anúncio no Google, visitar o site organicamente dias depois, receber um e-mail de CRM e só então converter por canal direto.

O modelo tradicional de Last Click atribui 100% do crédito ao último canal — ignorando todos os touchpoints anteriores que contribuíram para a decisão. Isso distorce a visão de ROI e pode levar ao sub-investimento em canais de awareness.

A solução foi construir uma plataforma que calculasse os dois modelos em paralelo, permitindo comparação direta e tomada de decisão mais informada sobre alocação de budget.

// Problema

Dados de conversão espalhados em múltiplas fontes sem visão unificada da jornada do cliente. Dependência total de Last Click distorcia o ROI real de cada canal.

// Solução

Pipeline de dados centralizado com modelo Markov para redistribuição estatística de crédito entre canais, com dashboards comparativos por produto financeiro.

// Impacto

Redistribuição do budget de mídia baseada em dados, com evidência de canais subestimados no modelo Last Click. Visibilidade completa de toda a jornada de conversão.

Algoritmo Central

Atribuição via Cadeias de Markov

Como funciona o modelo
O modelo de Markov trata a jornada do cliente como uma sequência de estados (canais de mídia) com transições probabilísticas entre eles. O crédito de cada canal é calculado pelo seu "removal effect" — ou seja, quanto a taxa de conversão cai quando aquele canal é removido do grafo de transições. Isso distribui o crédito de forma proporcional à contribuição real de cada touchpoint.
// 01
Construção do Grafo
Cada jornada de cliente é mapeada como sequência de canais → conversão. Calcula-se a probabilidade de transição entre cada par de canais.
// 02
Cálculo de Conversão Base
Com o grafo completo, calcula-se a probabilidade de conversão total considerando todos os caminhos possíveis até o estado absorvente.
// 03
Removal Effect
Para cada canal C, recalcula-se a prob. de conversão sem C. O removal effect é a diferença entre a taxa com e sem o canal.
// 04
Redistribuição
O crédito total (contratos/receita) é redistribuído proporcionalmente ao removal effect de cada canal, normalizando para 100%.
