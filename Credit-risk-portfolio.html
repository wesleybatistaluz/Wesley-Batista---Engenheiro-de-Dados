01 — O Problema

Por que um modelo de crédito?

Instituições financeiras enfrentam o desafio de avaliar risco de crédito de forma rápida e consistente. Decisões manuais são lentas, inconsistentes e suscetíveis a viés.

⚖️

Dados desbalanceados

Apenas 6.6% dos clientes são inadimplentes. Modelos comuns ignoram a minoria e erram onde mais importa — na detecção do risco.

📊

Muitas variáveis correlacionadas

Histórico de atrasos, renda, dependentes e uso do crédito interagem de forma não-linear. Regressão logística simples não captura isso.

🚀

Decisão em tempo real

O modelo precisa retornar uma decisão em milissegundos via API REST, consumível por qualquer sistema de crédito.

04 — Modelagem

XGBoost + SMOTE

Combinação de técnicas para lidar com o desbalanceamento e capturar padrões não-lineares nos dados.

🌲 XGBoost

Gradient boosting baseado em árvores de decisão. Captura interações não-lineares entre variáveis sem necessidade de normalização manual.

Hiperparâmetros

n_estimators=300

learning_rate=0.05

max_depth=5

⚖️ SMOTE

Synthetic Minority Oversampling Technique. Gera exemplos sintéticos da classe minoritária (inadimplentes) para balancear o treino.

Por que não peso de classe?

SMOTE cria vizinhos interpolados no espaço de features — gera representações mais ricas da minoria que simples oversampling.

🔄 Cross-Validation

Stratified K-Fold com 5 folds garante que cada fold mantém a proporção de classes originais, evitando vazamento de dados.

Resultado

ROC-AUC 0.8277

± 0.0059 (estável)

src/models/train.py — Pipeline Principal
pipeline = ImbPipeline([ ("smote", SMOTE(random_state=42)), ("scaler", StandardScaler()), ("model", XGBClassifier( n_estimators=300, max_depth=5, learning_rate=0.05, subsample=0.8, colsample_bytree=0.8, )) ]) # Cross-validation estratificado scores = cross_val_score( pipeline, X_train, y_train, cv=StratifiedKFold(n_splits=5), scoring="roc_auc" ) # ROC-AUC médio: 0.8277 ± 0.0059
