01 — Arquitetura

Como o pipeline funciona

Cada módulo é independente e testável. O Airflow orquestra a execução, garante retry automático em caso de falha e agenda a execução diária.

E
📥

Extract

Gera dados de vendas simulados com a biblioteca Faker — produtos, categorias, regiões, preços e descontos realistas. Em produção, aqui entraria a leitura de uma API, S3, banco legado ou arquivo CSV.

src/extract/extract.py
T
⚙️

Transform

Limpa duplicatas, remove outliers, trata nulos e cria colunas calculadas: receita bruta, líquida, desconto aplicado, segmentação de ticket e features de tempo (mês, trimestre, semana).

src/transform/transform.py
L
💾

Load

Carrega os dados no PostgreSQL usando upsert inteligente — nunca duplica registros. Mantém staging para rastreabilidade, fact table para análise e dimensões de produto e região.

src/load/load.py
R
📊

Report

Gera 6 gráficos com matplotlib/seaborn, um dashboard HTML totalmente interativo com Plotly e um relatório PDF completo com capa, KPIs e tabelas usando fpdf2.

src/reports/generate_report.py

06 — Como Rodar

Passo a passo

Apenas Docker Desktop é necessário. Todo o ambiente sobe automaticamente com um único comando.

1
Instale o Docker Desktop
Baixe em docker.com/products/docker-desktop e instale. Abra o Docker Desktop e aguarde a baleia aparecer na barra de tarefas.
2
Clone o repositório e suba os containers
Abra o terminal na pasta do projeto e execute:
docker compose up -d
Aguarde ~2-3 minutos para o Airflow inicializar completamente.
3
Instale as dependências nos containers
Execute nos dois containers:
docker exec -u airflow etl-pipeline-airflow-webserver-1 python -m pip install pandas matplotlib seaborn plotly fpdf2 sqlalchemy psycopg2-binary faker openpyxl
docker exec -u airflow etl-pipeline-airflow-scheduler-1 python -m pip install pandas matplotlib seaborn plotly fpdf2 sqlalchemy psycopg2-binary faker openpyxl
4
Acesse o Airflow UI e execute o DAG
Abra http://localhost:8080 — login: admin / senha: admin. Ative o DAG etl_sales_pipeline e clique em ▶ Trigger DAG.
5
Baixe os relatórios após tudo verde
Quando todas as 6 tasks ficarem verdes (~3-5 min):
docker cp etl-pipeline-airflow-scheduler-1:/opt/airflow/reports/output/dashboard.html .
docker cp etl-pipeline-airflow-scheduler-1:/opt/airflow/reports/output/relatorio_vendas.pdf .
6
Consulte o banco de dados
Para ver os dados carregados no PostgreSQL:
docker exec -it etl-pipeline-postgres-1 psql -U airflow -d airflow
Depois rode: SELECT COUNT(*) FROM warehouse.fact_sales;
